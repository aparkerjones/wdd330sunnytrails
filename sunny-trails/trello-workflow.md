# Trello Workflow (Week 5)

## Lists

- Backlog Week 5
- Backlog Week 6
- Backlog Week 7
- In Progress

## Label Meanings

- planning: scope, docs, setup
- design: wireframes, visual tokens, layout
- api: service integration and data mapping
- ui: rendering and interaction views
- itinerary: itinerary CRUD and summary
- storage: local storage persistence
- testing: unit, QA, cross-browser checks
- deploy: deployment and release tasks

## Working Rules

1. Move only 3-5 cards into `In Progress` at a time.
2. Add a checklist to every implementation card.
3. Use comments to record blockers and decisions.
4. Move to `Done` only after local verification.
